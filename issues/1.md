## Root Cause
In `.github/workflows/circular-dependency-check.yml` at lines 98-99, the script attempts to access `project.extras`, but `ProjectInfo` returned by `Pkg.project()` does not have an `extras` field in recent Julia/Pkg versions.

## Solution
Replace the problematic code in the "Generate dependency report" step with direct TOML parsing:

```julia

using TOML;

println("="^50);

$(project.name) v$(project.version)");

deps = length(project.dependencies);
weakdeps = haskey(toml, "weakdeps") ? length(toml["weakdeps"]) : 0;
extensions = haskey(toml, "extensions") ? length(toml["extensions"]) : 0;

println("ðŸ“Š Dependencies: $deps regular, $weakdeps weak, $extensions extensions");
```
