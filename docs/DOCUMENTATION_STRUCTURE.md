# Documentation Structure

## Directory Organization

This directory contains all project documentation organized by category:

### `/docs/hpc/`
HPC cluster deployment and infrastructure documentation:
- Deployment guides and strategies
- Bundle creation and management
- Direct node access procedures
- Falcon cluster usage

### `/docs/project-management/`
Project management and tracking documentation:
- GitLab issue tracking
- Phase implementation plans
- Sprint and milestone documentation
- Task extraction and classification

### `/docs/archive/`
Historical and superseded documentation:
- Legacy approaches that have been replaced
- Old infrastructure documentation
- Completed phase summaries

### `/docs/development/`
Active development guides and references:
- Package dependency information
- Development workflows
- Current implementation strategies

## Root Directory Files

The following files remain in the root directory:
- `README.md` - Project overview and getting started
- `CHANGELOG.md` - Project changelog (standard location)
- `CLAUDE.md` - Claude Code context and memory (required for AI assistance)
